1) أنشئ مشروع Supabase.
2) افتح SQL Editor والصق الملف: supabase_schema.sql ثم نفذه.
3) في Authentication عطّل تأكيد البريد مؤقتاً إن أردت دخولاً فورياً للاختبار.
4) افتح التطبيق واضغط: ربط Supabase.
5) ألصق Project URL و Anon Key.
6) أنشئ حساب مفتش من الواجهة أو عبر معالج البداية.
7) أضف الأساتذة من لوحة المفتش أو دع الأستاذ ينشئ حسابه ببريد المفتش.
8) رفع الملفات الحقيقي يتم إلى bucket اسمه reports.

مهم:
- هذه النسخة تحتاج استضافة http/https، وليس فتح الملف مباشرة عبر file://
- لو فعّلت Email confirmation، سيتوجب تأكيد البريد قبل الدخول.
- الصلاحيات هنا RLS على الجداول الأساسية، ويمكنك تشديد Storage policies أكثر لاحقاً حسب مسار الملفات.
